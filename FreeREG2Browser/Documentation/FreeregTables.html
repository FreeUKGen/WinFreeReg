FreeREG Tables

One significant change between the old and the new programs is connected with the way in which uploaded files are now validated by the new FreeREG server. Previously, there existed a state of anarchy in the way that Place and Church names were spelled, leading to multiple interpretations of names and subsequent confusion for researchers. With the advent of FreeREG/2 and WinFreeREG, only Place and Church names already known to the server will be accepted in all NEW transcription files.

To aid the user in adhering to these constraints, WinFreeREG will only allow files to be created using names that it has previously obtained from the server. To facilitate this, under the Transcription Data drop-down menu, there is an option that you can use to obtain this so-called 'fixed' data: FreeREG Tables.

Register Types
FreeREG actually defines what 2-character codes are acceptable when starting a new file. Again, the reasoning is one of consistency. The user has no way of adding entries to the table or removing any entries. This can only be achieved by downloading the table again.
Counties
FreeREG now defines which Counties can be selected. This is slightly different to the case in WinREG, where the list of acceptable Counties was effectively hard-coded into the program. Transcribing can't really start until the list of counties has been downloaded from the FreeREG server.
Place names
As stated above, transcribers can now only use those Placenames that have been defined by the FreeREG server. These have to be downloaded from the server before transcribing into new files can begin. Downloading is by County: so, select a county and click the Fetch button.
Church names
As with Placesnames, Church names are now defined on the FreeREG server, and need to be downloaded before transcribing begins. Churches are linked with Places: so, select the County and the Place name and click the Fetch button.
